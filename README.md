# Practice_2_Task_4

## Требования

Для запуска проекта необходимы:

- Node.js версии 24 или новее;
- npm;
- git для клонирования репозитория.

## Установка и запуск

Склонируйте репозиторий:

```powershell
git clone https://github.com/VyacheslavKokorin/Practice_2_Task_4.git
```

Перейдите в папку проекта:

```powershell
cd Practice_2_Task_4
```

Создайте файл с переменными окружения на основе примера:

```powershell
Copy-Item .env_example .env
```

Перейдите в папку приложения и установите зависимости:

```powershell
cd app
npm install
```

Запустите приложение:

```powershell
npm start
```

После запуска сайт будет доступен по адресу:

```text
http://localhost:3000
```

Файл базы данных SQLite создаётся автоматически при первом запуске.

Приложение необходимо запускать командой `npm start` из папки `app`, чтобы переменные из файла `.env` загрузились правильно.

## Вход администратора

Для проверки интерфейса администратора используется тестовая учётная запись:

```text
Email: admin@bookstore.ru
Пароль: admin123
```

## Анализ выполненной задачи

1. Для проекта был выбран Node.js и Express. Такой вариант позволяет создать серверную web-страницу без использования отдельных frontend-фреймворков.

2. Для хранения информации используется SQLite. База хранится в одном локальном файле и создаётся автоматически, поэтому пользователю не нужно отдельно устанавливать MySQL или запускать Docker.

3. В базе данных созданы связанные таблицы пользователей, книг, авторов, категорий, покупок, аренд и уведомлений. Связи через внешние ключи помогают не сохранять покупку или аренду для несуществующей книги.

4. В приложении реализована регистрация и авторизация. Пароли не хранятся в открытом виде, а перед сохранением хешируются с помощью bcrypt. После регистрации пользователь получает 2000 рублей на баланс.

5. Пользователь может открыть каталог, посмотреть отдельную книгу и отсортировать список по названию, автору, категории или году написания. Информация об авторах и категориях получается с помощью SQL-запросов с JOIN.

6. При покупке книги приложение проверяет доступность, баланс и наличие прошлой покупки. Списание денег и сохранение покупки выполняются в транзакции, чтобы при ошибке не потерять деньги пользователя.

7. Для аренды доступны сроки 2 недели, 1 месяц и 3 месяца. Цена аренды рассчитывается от стоимости книги, а дата окончания создаётся автоматически. Повторная активная аренда этой же книги запрещена.

8. Пользовательская страница «Мои книги» показывает баланс, покупки и действующие аренды. Просроченные аренды получают отдельный статус и перестают показываться как активные.

9. Интерфейс администратора защищён проверкой роли. Администратор может добавлять и удалять книги, а также менять их цену, статус и доступность.

10. Сервер автоматически проверяет сроки аренды. За три дня до окончания создаётся уведомление, которое пользователь видит на отдельной странице.

11. Для страниц используется общий HTML-шаблон и один CSS-файл. Благодаря этому навигация и оформление не повторяются отдельно в каждом маршруте.

## Выявленные ошибки и рекомендации

1. Сейчас сессии хранятся в памяти процесса. Для большого проекта лучше подключить отдельное хранилище сессий, чтобы пользователи не выходили из аккаунта после перезапуска сервера.

2. Проверка данных форм сделана только на базовом уровне. В дальнейшем стоит добавить ограничения длины имени и пароля, более строгую проверку email и очистку введённого текста.

3. HTML создаётся с помощью шаблонных строк. Если магазин будет принимать произвольный текст от пользователей, перед выводом его нужно экранировать для защиты от вставки HTML и JavaScript.

4. Удаление книги также удаляет связанные покупки и аренды. В реальном магазине лучше не удалять такие книги полностью, а менять их статус на архивный для сохранения истории операций.

5. Напоминания показываются только внутри сайта. Для улучшения проекта можно добавить отправку сообщений по email, но для этого потребуется отдельный почтовый сервис.

6. Цены хранятся в типе REAL. Для более точной работы с деньгами можно хранить сумму в копейках как целое число, чтобы избежать ошибок округления.

7. Автоматическая проверка аренды выполняется внутри web-сервера. При дальнейшем развитии её лучше вынести в отдельную задачу по расписанию, которая не зависит от перезапуска приложения.
